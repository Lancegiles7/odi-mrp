Behaviour

  • Save-on-blur — same as Prod inputs. Enter key also blurs/saves.
  • Editing affects balance live — Balance columns recompute as you type so the red shortfalls move with the opening stock.
  • Empty input = no override — clearing the field sets opening_stock_override = NULL, so the future on-hand inventory value will take over automatically.
  • Column header hint — small "editable" tag under the "Opening" label so it's obvious on first glance.
  • Scope — same permissions as the Prod cells (no admin-only gate needed beyond existing auth).

Implementation (for reference)

  • New server action updateOpeningStock(productId, units | null) in app/(dashboard)/production/actions.ts.
  • ProductionRow gains local state for opening + input; onBlur calls the action.
  • Balance calc already keys off opening → live recompute comes for free.
  • No schema changes — column already exists.
